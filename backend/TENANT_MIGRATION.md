## 🔄 Key Differences: Go vs Elixir

| Aspect | Go | Elixir |
|--------|-----|--------|
| **ORM** | GORM | Ecto |
| **ID Generation** | `uuidx.MustNewV7()` | Ecto auto-generates UUIDs |
| **Timestamps** | Manual `time.Now().UTC()` | Ecto `timestamps()` macro |
| **Validation** | Custom code | Ecto changesets |
| **Queries** | GORM methods | Ecto.Query DSL |
| **Transactions** | `db.InTx()` | `Repo.transaction()` |
| **Errors** | `app.Error()` | Pattern matching `{:ok, _}` / `{:error, _}` |

## 🎯 Usage Examples

### Create a Business:
```elixir
Easy.Tenant.create_business(%{
  handle: "acme_corp",
  name: "Acme Corporation",
  about: "We make widgets",
  owner_user_id: user_id
})
```

### Get Plans:
```elixir
plans = Easy.Tenant.list_plans()
default_plan = Easy.Tenant.get_default_plan()
```

### Create Subscription:
```elixir
Easy.Tenant.create_subscription(%{
  business_id: business_id,
  plan_id: plan_id,
  max_active_clients: 10,
  status: 4  # active
})
```

## ⚠️ Important Notes

1. **UUIDs**: Elixir uses `:binary_id` type, auto-generated by Ecto
2. **Timestamps**: Ecto uses `inserted_at`/`updated_at` instead of `created_at`/`updated_at` in column names, but maps to the same database columns
3. **Enums**: Subscription status stored as integers (1-5), use `Subscription.status_values()` for mapping
4. **Preloading**: Always preload associations (e.g., `Plan` has_many `Prices`)
