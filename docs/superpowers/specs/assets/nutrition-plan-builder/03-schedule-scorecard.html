Schedule options — case by case

Five plan shapes a coach actually builds. Verdict each, tally at the end, then a hybrid that may beat all three. Tap the winner.

1 · Same meals every day A

Standard cut/bulk — identical 6 slots all week. Most common by far.

A · template win

Fill 6 slots once. Done. Zero repetition.

B · grid

Type the same meal into 7 columns × 6 rows. Tedious.

C · day accordion

Rebuild all 7 days by hand. Worst.

2 · Weekday / weekend split A

Strict Mon–Fri, relaxed Sat–Sun. Very common.

A · template win

Daily template + override Sat & Sun only. Two overrides.

B · grid

Visible, but edit 12 weekend cells individually.

C · day accordion

Edit each of 7 days.

3 · Fully different every day B

High-touch, rotating meals daily. Rare.

A · template

Override all 7 days — the template buys nothing here.

B · grid win

Built for it — fill every cell, see it all.

C · day accordion

Works, but no overview while editing.

4 · Scan the whole week at a glance B

"Is Thursday dinner set? Any gaps?"

A · template

Per-day hidden behind "Customize" until opened.

B · grid win

Everything on one screen; gaps obvious.

C · day accordion

One day at a time.

5 · Mobile width & tap size A

One-page builder, phone.

A · template win

Full-width single column, big taps.

B · grid

42 tiny cells, horizontal scroll — the nesting/width trap.

C · day accordion

Fine width, but lots of tapping.

A · 3B · 2C · 0
Winner: a hybrid — A to edit, B as a read-only overview. Same move we made on the coach training builder (simple edit surface + read-only week projection). A owns the common cases and mobile; B owns whole-week scanning. Combine them: edit in the daily-template column, and a compact read-only week grid sits above/below for the at-a-glance view (tap a cell → jumps you to that day's override). C loses outright.
