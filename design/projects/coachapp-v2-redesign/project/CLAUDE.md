# CoachEZ — Project Context

Coaching SaaS (personal-training / fitness coaching) UI, built as Design Components (`.dc.html`).
Brand: **coachez** (renamed from "coacheasy"). Logo at `assets/icon-192.png`, wordmark "coachez".

## Design system ("Focus" language)
- Fonts: **Space Grotesk** for headings/numbers, system sans for body.
- Accent blue `#0091FF` (Apple-ish system blue). Ink `#18181B`. Muted `#71717A` / `#A1A1AA`. Hairline `#ECEEF2`/`#F1F4F8`. Surface off-white `#F7F8FA`.
- Sidebar (app chrome) is dark `#18181B`; active nav item is accent-blue filled.
- Rounded corners, hairline borders, soft shadows, `fadeUp`/spring animations. Inline styles only (DC rules).
- Aesthetic target the user keeps pushing toward: **clean, calm, Apple-like** — minimal color, generous spacing, iOS segmented controls, frosted overlays. Avoid clutter and "loaded with everything".

## Key runtime gotchas
- Programmatic `.click()` in eval sometimes doesn't fire React handlers reliably; multiple setStates in one eval don't re-render mid-eval — verify each state change in its own eval call.
- SVG `<text>` renders 0×0 here — use absolutely-positioned HTML for any chart/axis labels.
- `run_script` refuses writes that shrink a file >50% (write to temp + move instead).
- Overlays/menus that bleed into background: fix by raising the open element's `z-index` (position:relative; z-index) and/or a fixed blurred backdrop div.

## Working style the user prefers
- Very concise. Small targeted edits, don't redesign unasked. Iterates a LOT on spacing/visual polish ("too congested", "make it Apple-like", "undo it"). Frequently wants overlays to float above a blurred backdrop rather than overlap content.
